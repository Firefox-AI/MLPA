App Attest

iOS App Attest verification flow: (1) GET /verify/challenge to obtain a challenge, (2) POST /verify/attest with a JWT containing the attestation object. Use the attested key for subsequent requests to /v1/chat/completions with use-app-attest header.

Get attestation challenge

Generate a challenge for iOS App Attest. The client uses this challenge when creating an attestation key. Call this first, then POST the attestation to /verify/attest.

query Parameters
key_id_b64
required
string (Key Id B64)

Base64-encoded attestation key ID from the iOS device

Responses

/verify/challenge

Response samples

  • 200
  • 422
Content type
application/json
{
  • "challenge": "string"
}

Verify attestation

Submit the App Attest attestation object for verification. Authorization must be a Bearer JWT whose payload matches the AttestationAuth schema. Use the challenge from GET /verify/challenge.

header Parameters
authorization
required
string (Authorization)

Bearer JWT with payload matching AttestationAuth schema

Use-Qa-Certificates (boolean) or Use-Qa-Certificates (null) (Use-Qa-Certificates)

Use QA certificates (set to true for development/sandbox testing)
